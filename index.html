PRD Writer Agent (Enhanced v2)
How it works: Turns your product ideas into detailed project plans through conversation. Asks smart questions to understand what you need, then creates a complete specification document that development teams can use to build the product. Includes everything from features and timelines to risks and success metrics.
Input Requirements
  • Conversation depth: Works with brief ideas (2-3 sentences) or detailed enterprise discussions
  • Context needed: Problem description, target users, business goals, stakeholder landscape
  • Interactive mode: Agent asks comprehensive clarifying questions with deeper context exploration
What It Creates
  • Executive Summary with vision, objectives, expected outcomes, stakeholder buy-in
  • Core Feature Areas with detailed user stories, requirements, acceptance criteria
  • Technical Architecture with implementation strategy and system integrations
  • Success Metrics, comprehensive risk management, detailed timeline estimates
  • Additional sections: Go-to-market strategy, rollout phases, training plans
  • Typical output: 40-60 pages, enterprise production-ready
Special Capabilities
  • Enhanced PM frameworks (Jobs-to-be-Done, OKRs, Impact Mapping)
  • Generates realistic timelines with resource allocation and team capacity planning
  • Comprehensive risk identification with mitigation strategies
  • Stakeholder analysis and change management considerations
  • Enterprise-grade quality for complex multi-team projects
Feature Extractor Agent
How it works: Breaks down the project plan into individual features that can be built separately. Estimates how long each feature will take and determines which features need to be built first. Creates a prioritized list showing what to build when.
Input Requirements
  • Source: Complete PRD document (any format: MD, PDF, TXT)
  • Minimum sections: Feature areas or requirements list
  • Optional: Success metrics help prioritization
What It Creates
  • Structured feature list with unique IDs
  • Story point estimates (Fibonacci scale: 1, 2, 3, 5, 8, 13, 21)
  • Priority ratings (P0/Critical, P1/High, P2/Medium, P3/Low)
  • Implementation phases based on dependencies
  • Effort breakdown by phase with timeline estimates
Special Capabilities
  • Identifies feature dependencies automatically
  • Balances technical feasibility vs. business value
  • Suggests MVP scope vs. future enhancements
Feature PRD Writer Agent
How it works: Takes each feature and creates a detailed specification for it. Includes user stories, what success looks like, technical requirements, and what could go wrong. Gives developers everything they need to build that specific feature.
Input Requirements
  • Feature description: 1-3 paragraphs minimum
  • Context: Project PRD for background (optional but recommended)
  • Batch processing: Can process entire feature list at once
What It Creates
  • User stories in standard format (As a... I want... So that...)
  • Detailed acceptance criteria (Given/When/Then scenarios)
  • Technical requirements (APIs, data models, integrations)
  • Edge cases and error handling scenarios
  • UX considerations and accessibility requirements
  • Typical output: 8-12 pages per feature, engineering-ready
Special Capabilities
  • Generates testable acceptance criteria automatically
  • Considers cross-feature dependencies and data flow
  • Includes security, performance, and scalability considerations
  • Ready for JIRA ticket conversion
